# Изовисты (Isovist) и поведение посетителей

Все файлы анализа по изовистам лежат в этой папке: `model_search/Isovist/`.

## Что сделано

- **План стен** (этаж 0 из `NMFA_3floors_plan.json`) используется для ray-casting из каждой точки траектории.
- Для каждой точки считаются:
  - **isovist_area** — площадь видимой области (в единицах плана²);
  - **isovist_perimeter** — длина видимого периметра (в единицах плана).
- По траектории агрегируются: `isovist_area_mean/min/max`, `isovist_perimeter_mean/min/max`.
- Результат объединяется с `openness_and_movement.csv` → `isovist_and_movement.csv`.
- В подпапке **analysis/** — корреляции изовист–движение, scatter isovist_area vs speed.

**Запуск** (из корня проекта):
```bash
python "model_search/Isovist/isovist_visibility_analysis.py"
# Если точек много — ускорить: каждая 5-я точка по траектории
python "model_search/Isovist/isovist_visibility_analysis.py" --sample 5
```

Предварительно должен быть выполнен этап 1 в `Openness and size of the space` (файлы `points_with_zone_and_width.csv` и `openness_and_movement.csv`). Если их нет, скрипт возьмёт точки из `analysis_results/floor0_trajectories_with_features.csv`.

**Файлы в этой папке:**
- `isovist_visibility_analysis.py` — расчёт изовистов и анализ
- `points_isovist_features.csv` — признаки по точкам (после запуска)
- `trajectory_isovist_features.csv` — агрегация по траекториям
- `isovist_and_movement.csv` — изовист + движение для корреляций
- `analysis/` — тепловые карты корреляций, scatter-графики

## Идеи расширения (что ещё можно добавить)

1. **Центры зон**  
   Считать изовист не только по точкам траектории, но и по центроидам зон (0–15) — тогда у каждой зоны будет своя «видимость», и можно смотреть корреляцию «видимость зоны» ↔ посещаемость/остановки.

2. **Что видно из точки**
   - **Картины**: для каждого центра экспоната (из плана или `floor0_attractions.json`) проверять, попадает ли он внутрь полигона изовиста → признак «число видимых картин», «видна ли цельовая картина».
   - **Залы/зоны**: доля площади каждой зоны, попадающей в изовист, или бинарно «видна ли зона N».
   - **Окна**: отрезки из `floor0_windows.json` — пересекает ли луч окно (не стену) → «видимость окна», «число видимых окон».
   - **Лестницы/переходы**: если в данных есть полигоны лестниц или дверей — попадание в изовист даёт признаки «видна ли лестница», «сколько выходов видно».

3. **Сложность панорамы**
   - **Число «поворотов»**: по полигону изовиста — число значимых изгибов контура (упрощённый полигон, потом подсчёт вершин или углов).
   - **Компактность**: perimeter²/area или форма фактор (круг = 1, вытянутый/сложный — больше) — насколько «простая» vs «сложная» видимая панорама.

4. **Видимость вперёд по пути**
   - По направлению движения в точке: сектор изовиста впереди (например ±30°) — площадь/длина видимого контура только в этом секторе. Можно связать с скоростью («где видно далеко — идут быстрее»).

5. **Сэмплирование точек**
   - Если точек много (десятки тысяч), можно считать изовист не для каждой, а для каждой K-й точки траектории или по сетке, чтобы ускорить расчёт и потом интерполировать/агрегировать.

Реализацию удобно делать поверх уже сохранённого полигона изовиста (вершины по углам) в `compute_isovist_at_point`, возвращая не только area/perimeter, но и списки координат границы и, при необходимости, пересечения с заданными отрезками/полигонами (окна, центры картин).
