# Окна и естественное освещение — гипотезы поведения

Разрывы в отрисовке outline этажа считаются окнами (естественное освещение). Скрипт проверяет связь «близость к окнам» с метриками движения (скорость, остановки, длительность и т.д.).

## Автоопределение окон

**detect_windows_from_plan.py** — по плану этажа 0 находит разрывы между коллинеарными стенами (зазоры заданного размера) и записывает их в **floor0_windows.json**. Запуск:
```bash
python "model_search/Windows/detect_windows_from_plan.py"
```
Пороги: `MIN_WINDOW_GAP`, `MAX_WINDOW_GAP`, `ANGLE_TOL_DEG`, `PERP_DIST_TOL` в начале скрипта.

## Что сделано

1. **floor0_windows.json** — список сегментов окон (разрывов). Каждый сегмент задаёт отрезок стены, где в контуре этажа есть разрыв (окно). Формат одного сегмента:
   ```json
   { "id": "win_1", "position": [ {"x": 100, "y": 4500}, {"x": 150, "y": 4500} ] }
   ```
   Координаты **в той же системе**, что в `NMFA_3floors_plan.json` и в траекториях.

2. **windows_natural_light_analysis.py** — считает по точкам траекторий:
   - расстояние до ближайшего окна (`dist_window`);
   - долю времени «рядом с окном» (`pct_near_window` — в радиусе `NEAR_WINDOW_RADIUS`);
   объединяет с `openness_and_movement.csv` из папки Openness, строит корреляции и графики (корреляционная матрица, scatter, boxplot по квартилям). Все результаты сохраняются в папке **Windows** (и подпапке **analysis/**).

## Как заполнить сегменты окон

- По **плану/визуализации**: там, где в outline этажа есть разрыв (или пунктир) — это окно. Возьмите координаты двух концов этого разрыва в системе плана (как у стен в `NMFA_3floors_plan.json`: `position[0].x, position[0].y` и `position[1].x, position[1].y`) и добавьте в `floor0_windows.json` в массив `segments`.
- Если контур экспортируется из Unity (например, `floor_outline` в JSON): последовательные точки контура с очень большим скачком между ними могут означать разрыв (окно). Можно выгрузить контур, найти такие скачки и записать соответствующие отрезки в `floor0_windows.json`.

## Запуск

1. Заполните `floor0_windows.json` сегментами окон (хотя бы один).
2. Убедитесь, что уже выполнены этапы открытости: `model_search/Openness and size of the space/openness_space_analysis.py` (появится `openness_and_movement.csv` и при необходимости `points_with_zone_and_width.csv`).
3. Из корня проекта:
   ```bash
   python "model_search/Windows/windows_natural_light_analysis.py"
   ```

Результаты: в папке **Windows**: `windows_and_movement.csv`, `trajectory_window_features.csv`; в **Windows/analysis/** — графики и матрица корреляций (`windows_*`).

## Настройка

В скрипте константа **NEAR_WINDOW_RADIUS** (по умолчанию 250 в единицах плана) задаёт, что считать «рядом с окном». При необходимости измените её под масштаб этажа.
