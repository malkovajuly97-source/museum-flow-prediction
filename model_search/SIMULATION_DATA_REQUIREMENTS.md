# Требования к данным из симуляции для сравнения с BIRD dataset

Чтобы вычислить все параметры (ranking, transition matrix, density, ToP, stop duration) и сравнить с датасетом BIRD, симуляция должна экспортировать траектории в следующем формате.

---

## Минимальный формат траекторий

### Структура данных

**Один CSV на траекторию** (или один объединённый CSV с идентификатором траектории).

### Обязательные колонки

| Колонка      | Тип   | Описание |
|--------------|--------|----------|
| `timestamp`  | float  | Время в **секундах** (как в BIRD) |
| `x`          | float  | Координата X |
| `y`          | float  | Координата Y |
| `floorNumber`| int    | Номер этажа (0 для этажа 0) |

### Пример строки

```csv
timestamp,x,y,floorNumber
0.0,2597.93,4596.40,0
2.0,2577.81,4355.46,0
4.0,2542.50,4148.36,0
```

---

## Система координат

**Критично:** координаты `x`, `y` должны быть в **той же системе**, что и BIRD dataset и DXF `floor0_paintings_areas.dxf`.

- BIRD использует сырые координаты (x, y); скрипты internally применяют `SCALE_FACTOR = 55.07 / 5401` для перевода в метры.
- Зоны 0–15 в DXF заданы в этой же системе.
- Unity / другая симуляция: нужно экспортировать в координаты плана этажа (референс: `NMFA_3floors_plan.json` или DXF).

---

## Какие параметры что используют

| Параметр            | Нужные поля         | Доп. данные |
|---------------------|---------------------|-------------|
| **Ranking**         | x, y, floorNumber   | DXF зоны 0–15 |
| **Transition matrix** | x, y, floorNumber, timestamp | DXF зоны 0–15 |
| **Density**         | x, y, floorNumber   | Сетка 1×1 м по min/max |
| **Time of Presence** | x, y, timestamp, floorNumber | Сетка 1×1 м |
| **Stop duration**   | x, y, timestamp, floorNumber | Сетка 1×1 м |

---

## Интервал дискретизации

- BIRD: точки с интервалом ~2 сек (timestamp: 0, 2, 4, 6, …).
- Для корректного сравнения stop duration и ToP желательно использовать тот же интервал (2 сек) в симуляции.

---

## Структура файлов (рекомендуемая)

Симуляция может экспортировать траектории в папку, например:

```
simulation_trajectories/
  001_traj.csv
  002_traj.csv
  ...
```

Каждый файл — одна траектория ( один агент). Формат — тот же, что у BIRD:

```
timestamp,x,y,floorNumber
0.0,2500.0,4000.0,0
2.0,2505.0,4010.0,0
...
```

---

## Итог: что экспортировать из симуляции

1. **Траектории** — CSV с колонками `timestamp`, `x`, `y`, `floorNumber`.
2. **Координаты** — в системе плана этажа (совместимой с BIRD/DXF).
3. **Интервал** — желательно 2 сек между точками.
4. **Этаж** — для этажа 0 используйте `floorNumber = 0`.

После экспорта можно использовать те же скрипты (`room_popularity.py`, `transition_matrix.py`, `density.py`), указав папку с траекториями симуляции вместо `bird-dataset-main/data/normalized_trajectories`. Либо написать обёртку, которая принимает обе папки и сравнивает результаты.
